Components

CoralUI provides several components intended as building blocks for your application. Some components are only composed of HTML and CSS while others also require JavaScript. Let's take a deeper look at the mechanics of components that require JavaScript.

Instantiating a Component

Each component that relies on JavaScript has a "class" that manages the logic, DOM, and model for the component. For example, the Accordion component has a class that acts as a blueprint for all instances of Accordion. For each DOM element that represents a component, a single JavaScript instance must be created in order for the component as a whole to function properly. There are multiple ways these instances can be created as described below.

Instantiating through the DOM API

When your application's document is first loaded, CoralUI will automatically scan the document for elements with data-init attributes and create JavaScript component instances based on the attribute's value. For example, CoralUI will look for any element with the attribute of data-init="numberinput" and, for each element it finds, will automatically create an instance of the NumberInput component class. The JavaScript instance will then manage the DOM element that held the data-init attribute.

  If you add an element after the document as loaded, this process of scanning the document and instantiating objects will not happen again automatically. Instead, you will need to manually restart the process by triggering a cui-contentloaded event on an ancestor of the element:

                        $(document).trigger('cui-contentloaded');

Note that the data-init attribute is only useful when automatically instantiating components using the DOM API; it provides no other purpose.

Instantiating using a JavaScript constructor

You can also instantiate the component directly through JavaScript by using the component's class constructor.

                        var numberInput = new CUI.NumberInput({ element:'#myNumberInput' });

Notice the JavaScript instance must be made aware of the DOM element that it should manage. It will accept a selector (as shown in the example), a DOM element, or a jQuery object.

To pass other options to the constructor, add them to the object being passed as an argument.

                        var numberInput = new CUI.NumberInput({ element:'#myNumberInput', max: 200 });

Retrieving a component's JavaScript instance

When a component is instantiated, the JavaScript instance is placed in a data store for its associated element using jQuery.data. Likewise, at any time you can retrieve this instance from the element's data store by using jQuery.data.

                        var numberInput = $('#my-number-input').data('numberInput');

The string passed to the method is the decapitalized class name. In this case, the class name is NumberInput. By lowercasing the first letter, we arrive at numberInput.

Programatically interacting with a component

Once a component is instantiated, you may wish to programatically interact with the component by calling methods on its JavaScript instance. For example, you may want to programatically add a new tab to a TabPanel. To do this, retrieve the JavaScript instance then call the appropriate method as documented.

                        var tabPanel = $('#my-tab-panel').data('tabs');
                        tabPanel.addItem({
                          tabContent: 'Fruits',
                          panelContent: 'Apples, Bananas, Oranges'
                        });

If what you are changing is an option that can also be passed through the constructor, you will need to instead use the set() method. For example, after a TabPanel has been instantiated, the third tab can be activated by calling the set() method passing active for the name of the option and 2 for the 0-based index of the tab to be activated.

                        var tabPanel = $('#my-tab-panel').data('tabs');
                        tabPanel.set('active', 2);

The nuance between when set() must be called instead of other methods like addItem() can be frustrating to consumers. We understand and will be simplifying this in a future release. Please see the examples in the component documentation for specifics on which methods to use in the meantime.

Responding to user interaction

When a user interacts with a component, the component will often trigger useful events from its DOM element. For example, you may want to react to a selection on the Select component. This is done by adding an event listener to the component's element.

                        $('#my-select').on('selected', function(event) {
                          console.log(event.selected);
                        }

Please see the component documentation for which events each component triggers.
