Page

Current Version: 3.0.5-pre
Related Components: ShellWidget, Shell
Github: coralui-component-shell

The page widget orchestrates the shell's black bar, outer rail, inner rail, action bar, and footer. In the absence of any mark-up for the aforementioned components, the Page widget can generate them. By default, the widget doesn't generate any widgets, but there are several options that start with 'generate' that when set to 'true' will result in the corresponding widget to be generated.

CUI.Page is a controller. Together with CUI.Shell, it is the only widget within the shell that is aware of any other shell widgets. The remainder of the widgets doesn't know about the outside world. CUI.Page listens to inner- and outer rail toggles being clicked, setting the rail states accordingly, and vice versa. When there is limited horizontal space available, the controller will make sure to not show two rails at the same time. Last, CUI.Page will add or remove some styles to or from its children depending on what widgets are present at a given time. The absence of a black bar or action bar may for example require a number of styles to be removed in order for a page to lay out correctly.

The page widget is expected to be the child element of the shell widget. Because of this, iframes have been used in the examples below.

Page JSDoc

Class Description: The CUI.Page widget manages the appearance of the nav rail, inner rail, black bar, action bar and the footer.

Code

Instantiate with Class

var page = new CUI.Page({
    element: '#myPage'
});

Instantiate with jQuery

$('#myPage').page();
