UserAccountPopover JavaScript

The UserAccountPopover is one of many components that are specific to Digital Marketing. It can be instantiated using a data-init flag or by invoking JavaScript directly.

Default Example - No User

NOTE: If no user object is set the popover will just display a spinner unless markup is provided by the server. This is on purpose because it is assuming that updateUser will be called when a valid user object exists. This is particularly useful if you have your own custom way of loading in a user. If you want to create a popover without providing a user please provide all of your own markup.

 JSDoc


    
Class Description: This widget controls the UserAccountPopover component

This class will most likely be used in conjunction with the BlackBar component but can be initialized
elsewhere if desired. The data that is passed into this class matches the format defined by the marketing cloud
unified header. Specifically the API to get the user /content/mac/default/header/user.jsonp. If you have set the
active org id (data-active-org-id="customid@AdobeOrg") and set hasImsSession (data-has-ims-session="true")
on the shell component then all data will be loaded from the marketing cloud api's.
