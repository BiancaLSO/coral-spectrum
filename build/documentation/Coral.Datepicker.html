Coral.Datepicker

Current Version: 3.0.0
Github: coralui-component-datepicker

The Datepicker component provides desktop and touch friendly for both date and time selection. The component leverages moment.js for parsing, manipulating, and formatting dates.

Instantiation

Using the new operator

The component can be instantiated using the new operator. This allows direct manipulation of the methods and properties. 
var datepicker = new Coral.Datepicker();

Creating DOM

Instantiation using createElement is done using coral-datepicker as the element to construct. 
var datepicker = document.createElement('coral-datepicker');

Markup

The component is instantiated using the coral-datepicker tag. Check the attributes section for a complete list of the accepted attributes. 
<coral-datepicker></coral-datepicker>

mode

                  Datepicker can show the user its own (Coral) UI, or revert to the client browser's native
                  input element.
                  When the mode attribute is set to auto (as it is by default)
                  native input elements are used only when the client is detected to be on a mobile device (either
                  Android or iOS at this time). Otherwise Coral UI is shown.

                    When the mode attribute is set to coral, Coral UI components are shown,
                    always.
                    
                  

                      
value

                  Current value of the component, expressed as a string, formatted in accordance to the set
                  valueFormat. This is the string that will be send to the server when the form
                  is submitted.
                  When set to "today", the value will be transformed to the current day.

valueformat
The format used on parsing value into a Date value and vice verse.
                      
                      

                      
Show Markup
                          

                      
max

                  Maximum date that the calendar will accept as valid. It must not be less than its minimum. If no max value is
                  required, it must be set to null. Expects a YYYY-MM-DD[T]HH:mmZ formatted string.
                  When set to "today", the value will be transformed to the current day.
                      
                      

                      
Show Markup
                          

                      
min

                  Minimum date that the NumberInput will accept as valid. It must not be greater than its maximum. If no min value
                  is required, it must be set to null. Expects a YYYY-MM-DD[T]HH:mmZ formatted string.
                  When set to "today", the value will be transformed to the current day.
                      
                      

                      
Show Markup
                          

                      
headformat
The format to use for the calendar month header.

disabled

                  Indicates that the component is not available for interaction. While disabled, the value can still be changed
                  programatically.
