Conventions

CoralUI follows SUIT conventions for CSS class. SUIT defines a way to differentiate between components, sub-components (descendents), and modifiers. Additionally, all classes are prefixed with 'coral' to elimininate conflicts with other CSS. 

                          class="coral-ComponentName-descendentName--modifierName"
                          
                          class="coral-Alert--large"
                          
                          class="coral-Alert-title"

Please see the SUIT documentation for more information.

Themes

CoralUI provides the ability to display components in a light and dark theme. Themes are specified via the coral--light and coral--dark class modifiers on an ancestor of a component.

NOTE: coral--light must be applied as the top-level theme class. This class serves as the 'opt-in' class that applies background, typography and color styles to elements within.

Utility Classes

            

            
CoralUI provides some CSS utility classes that can be applied to any DOM element. These classes follow the SUIT convention of a u- prefix.

                
                

                      
                        class="u-coral-clearFix"
                        

                

                
                

                      
                        class="u-coral-screenReaderOnly"
                        

                

            
Please see coralui-core's utilities directory for a list of available classes.

          

          

            

                  
Opt-in Styling

            

            
Compatibility with consumer code is a primary goal for CoralUI and is attained using an opt-in approach for styling. As such, elements will typically not receive CoralUI styles until a CoralUI-specific class name (e.g. coral-* has been added to the element, or an ancestor element in cases where the styles cascade.

            
Themes (which include backgrounds, typography, and colors), links, headings, and lists are areas where some consumers may incorrectly assume that styles will automatically be applied. Please review the respective documentation to see how styles are correctly applied in such cases.

          

          

            

                  
Browser consistency

            

            
Another primary goal for CoralUI is to provide a consistent experience across browsers. CoralUI comes bundled with normalize.css which makes browsers render all elements more consistently and only targets styles that need normalizing.  While CoralUI goes to great lengths to avoid conflicts with consumer code, normalize.css does automatically adjust some elements. Its effects are minimal, but please poke around your application after introducing CoralUI to ensure everything still renders as expected.
