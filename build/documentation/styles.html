Styles

Conventions

CoralUI follows SUIT conventions for CSS. Please see the SUIT documentation for more information.

Opt-in styling

Compatibility with consumer code is a primary goal for CoralUI and is attained using an opt-in approach for styling. As such, elements will typically not receive CoralUI styles until a CoralUI-specific class name has been added to the element (or an ancestor element in cases where the styles cascade). Themes (which include backgrounds, typography, and colors), links, headings, and lists are areas where some consumers may incorrectly assume that styles will automatically be applied. Please review the respective documentation to see how styles are correctly applied in such cases.

Browser consistency

Another primary goal for CoralUI is to provide a consistent experience across browsers. CoralUI comes bundled with normalize.css which makes browsers render all elements more consistently and only targets styles that need normalizing. While CoralUI goes to great lengths to avoid conflicts with consumer code, normalize.css does automatically adjust some elements. Its effects are minimal, but please poke around your application after introducing CoralUI to ensure everything still renders as expected.
