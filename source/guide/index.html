What is CoralUI?

CoralUI or CUI (pronounced "cooey") is an implementation of the CloudUI visual style from Adobe XD.

CoralUI's aim is to provide everything your product needs to adopt the CloudUI visual style.

Resolution Independence

CoralUI is resolution independent. By using elastic measurement units to define sizes (em and rem), CoralUI can be scaled to the perfect size for any display, regardless of size or pixel density.

Browser Support

CoralUI supports all modern browsers, including IE 9+, Firefox (latest), Chrome (latest), Safari (latest).

Prevalent mobile devices are also be supported, including Android 3+ and iOS 5+.

Widget Philosophy

All CoralUI widgets are guaranteed to fulfill 8 promises:

  1. Widgets will be available as a jQuery plugin: $('#myModal').modal()
  2. Widgets will be available as a class: myModal = new CUI.Modal({ element: '#myModal' }).
  3. Widget APIs will always be fully documented here.
  4. Widgets will always be easily extendable with CUI.Modal.extend().
  5. Widgets will progressively enhance existing markup, or render a template using passed options if no markup is present.
  6. Widgets will take advantage of declarative markup using data-* attributes.
  7. Widgets will work on both desktop and mobile.
  8. Widgets will be fully tested with automated unit tests.

  • finger Finger tabs: Useful for tabs that have long titles

Buttons & Controls

CUI provides many useful styles and widgets for buttons & controls.

Buttons

The <button> tag is styled automatically. You can apply the button CSS class to any element to style it as a button, but be sure to add role="button" for accessibility.

Variations

By default, buttons are gray. There are two CSS classes that can be added to change the look of the button:

  • primary A blue button, used to indicate primary actions such as OK or save
  • warning A red button, used to indicate actions that are potentially destructive, such as delete

Icons (WIP)

Icons can be applied to any element with one of the icon-* classes. As a general rule, to keep the application accessible, always label your icons with a text. There are different techniques in case you want to visually hide your label: you can either apply the hidden-accessible class to the label, or include your icons in an element with the class icons-group.

Note: The icons currently use an arbitrary letter from the alphabet. This will later be replaced with a custom font, an SVG graphic, or a sprite image. Currently, unless the icon is in a icons-group, the icon takes the width of the letter in the font, this is a temporary behavior.

Status Indicators

        
Status indicators provide a way of conveying the progress of an ongoing process or the result of a process that has finished.


        

          
Alert

          
Alerts indicate to the result of an operation or that a new information is present.

          
          
Variations

          
There are 5 different styles of alerts:

          
    
            
  • error Indicate that an operation has failed or that a critical problem is present
            
  • notice Indicate that an operation completed with warnings, or that a potential problem is present
            
  • success Indicate that an operation completed without errors or warnings
            
  • info Provide information to the user, such as indicating that new reports are available
            
  • help Provide guidance to the user, such as links to help documentation
          

          
          
Sizing

          
Alerts usually contain concise messages, but can accommodate larger messages when the large CSS class is added.

          
          
The <button class="close"> tag can be omitted if an alert should not be dismissible.

  • error Draw attention to critical problems
            
  • notice Draw attention to potential problems
            
  • success Draw attention to a success state
          

          
          
Arrow Direction

          
Tooltips can point different directions by applying the following CSS classes:

          
    
            
  • arrow-top Point upward
            
  • arrow-bottom Point downward
            
  • arrow-left Point left
            
  • arrow-right Point right

Progress Bar

          
Use a progress bar to indicate the progress of a process.

          
A <label> tag included immediately after the progress bar can be used to display the numeric value.


          

            

              

                

              

              
            

            
<div class="progress">
  <div class="bar" style="width: 40%;"></div>
</div>
<label>40%</label>
