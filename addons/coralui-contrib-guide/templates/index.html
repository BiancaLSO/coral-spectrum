What is CoralUI?

CoralUI (or CUI) is an implementation of the CloudUI visual style.

CoralUI's aim is to provide everything your product needs to adopt the CloudUI visual style.

Resolution Independence

CoralUI is resolution independent. By using elastic measurement units to define sizes (em and rem), CoralUI can be scaled to the perfect size for any display, regardless of size or pixel density.

Browser Support

CoralUI supports all modern browsers, including IE 9+, Firefox (latest), Chrome (latest), Safari (latest).

Prevalent mobile devices are also supported, including Android 3+ and iOS 5+.

Widget Philosophy

All CoralUI widgets are guaranteed to fulfill these promises:

  1. Widgets will be available as a jQuery plugin: $('#myModal').modal()
  2. Widgets will be available as a class: myModal = new CUI.Modal({ element: '#myModal' }).
  3. Widget APIs will always be fully documented here.
  4. Widgets will always be easily extendable with CUI.Modal.extend().
  5. Widgets will progressively enhance existing markup, or render a template using passed options if no markup is present.
  6. Widgets will take advantage of declarative markup using data-* attributes.
  7. Widgets will work on both desktop and mobile.
  8. Widgets will be fully tested with automated unit tests.

Framework Support

Coral UI can be used together with 3rd party frameworks but special care needs to be taken if dynamic insertion of DOM elements takes place. AngularJS will do this for example, on loading a view template.

Most Coral UI components are defined using CSS only, but some rely on JavaScript. These components (Tabs, Rail, CardView, etc.) listen for the page to complete loading after which they setup. When an element gets inserted into the DOM dynamically after the static HTML has already been loaded, then the setup needs to be done manually. This is done by triggering a cui-contentloaded event on the least containing node that got inserted dynamically:

$("#idOfLeastContainingElement").trigger("cui-contentloaded");

Icons

Icons can be added to a number of different elements by using the icon-* classes that correspond to the icon's filename.

An example of each of the icons available is shown in the Icon Browser

Usage Guidelines

Icons should be applied to three different tags:

  • <i>: Illustrative
  • <a>: Navigational
  • <button>: Actionable

Accessibility

If you want to show the text, add the withLabel CSS class.

When not using withLabel, be sure to still include text inside of the tag to indicate the icon's function. This text won't be visible, but will be readable by screen readers.

Sizing

When used without context, icons should be given a size:

  • xsmall 12x12
  • small 18x18
  • medium 24x24
  • large 32x32

Variations

Icons will change color appropriately when hovered or clicked based on context. However, you may force a particular color by applying one of the following CSS classes:

  • white White icon at 100% opacity
  • blue Blue icon at 100% opacity

Rail

The side panel used for metadata and feeds.

Positioning

The rail can be placed on the left or the right by applying the left or right class.

Sections

By using the <section> tag, the rail can contain multiple sections separated by a border.

The bare and thin CSS classes can be added to remove the border or padding on each section respectively.

Section headings should be contained within the <h4> tag.

A simple markup-driven tab bar used to separate sections of content.

Variations

In addition to the default tabs style, there are 3 variations that can be applied by using the following CSS:

Click the link for each description to view the variant below.

  • tabs: Used in a modal dialog or as sub-tabs. (Default)
  • tabs nav: For when tabs are used to divide content on a page level
  • tabs white: Used in a modal dialog or as sub-tabs

Additionally you can apply the class full-page in cases where you use the tabs in full-screen situations. This will give them some more margin to not stick to the borders of your screen.

Buttons

The <button> tag is styled automatically. You can apply the button CSS class to any element to style it as a button, but be sure to add role="button" for accessibility.

Variations

By default, buttons are gray. There are two CSS classes that can be added to change the look of the button:

  • primary A blue button, used to indicate primary actions such as OK or save
  • warning A red button, used to indicate actions that are potentially destructive, such as delete

The datepicker component. Initial input value should be in the following format:

  • Date: Y-m-d (e.g. 1987-04-06)

  • Time: h:m (e.g. 20:35)

Cards

                
There are many cards for displaying data. They are not always card shaped, but we still call them that.

                
Variations

                
    
                    
  • Asset Card: for a single item.
                    
  • Board Card: for a Tartan board.
                    
  • Collection Card: for a collection of items.
                    
  • Directory Card: for a directory of assets.
                    
  • Page Card: for a (web) page.
                    
  • Project Card: for a project (that contains an asset folder, 
                        a website and some collections).
                

                
Usage

                
The markup for a card can be displayed in grid view or in list view, depending if it is wrapped in an 
                    element with a grid or list class.

                
Some cards can be displayed as a stack (e.g. pages that contain sub-pages, non-empty 
                    collections and non-empty projects).  To display these cards as a stack, simply add the 
                    stack class to the card.

                
If a card has to be displayed in a list view, it should contain the following elements:

                    <i class="select"></i><i class="move"></i>. 
If the items cannot be viewed as a list, these elements can be ommitted.

                
To hide some elements in grid or in list view, the show-grid, respectivly 
                    the show-list classes can be used.

                
When an image should be optimized or swapped when changing between grid and list view, simply place both 
                    images into a <span class="image"> wrapper, setting the class 
                    show-grid and show-list on each as appropriate. 
                    The backing JavaScript will prevent both images from being loaded until the user switches the
                    view state.

Modal

                    
Modals obscure the whole page and prevent interaction underneath them.

                    
Modals will progressively enhance existing markup or render their own template if no markup is present. See the documentation for more examples.


                    
Variations

                    
Modals can be styled to display alert messages by adding one of the following classes to the modal element or by setting the type option.

                    
    
                        
  • default Grey heading, no icon
                        
  • error Red heading, exclamation point icon
                        
  • notice Yellow heading, exclamation point icon
                        
  • success Green heading, check mark icon
                        
  • help Blue heading, question mark icon
                        
  • info Blue heading, info icon

Status Indicators

                
Status indicators provide a way of conveying the progress of an ongoing process or the result of 
                    a process that has finished.


                

                    
Alert

                    
Alerts indicate to the result of an operation or that a new information is present.

                    
Variations

                    
There are 5 different styles of alerts:

                    
    
                        
  • error Indicate that an operation has failed or that a 
                            critical problem is present.
                        
  • notice Indicate that an operation completed with warnings, 
                            or that a potential problem is present.
                        
  • success Indicate that an operation completed without errors 
                            or warnings.
                        
  • info Provide information to the user, such as indicating 
                            that new reports are available.
                        
  • help Provide guidance to the user, such as links to 
                            help documentation.
                    

                    
Sizing

                    
Alerts usually contain concise messages, but can accommodate larger messages when the 
                        large CSS class is added.

                    
The <button class="close"> tag can be omitted if an alert 
                        should not be dismissed.

Tooltip

                    
Tooltips give the user small pieces of information in context by pointing at objects on 
                        the page. Tooltips are often used to draw the user's attention to problems in form 
                        fields or suggested actions.

                    
                    
Variations

                    
Tooltips are blue by default to indicate they contain helpful information. When tooltips 
                        contain other information, such as errors or alerts, the following CSS classes can be applied:

                    
    
                        
  • error Draw attention to critical problems
                        
  • notice Draw attention to potential problems
                        
  • success Draw attention to a success state
                    

                    
                    
Arrow Direction

                    
Tooltips can point different directions by applying the following CSS classes:

                    
    
                        
  • arrow-top Point upward
                        
  • arrow-bottom Point downward
                        
  • arrow-left Point left
                        
  • arrow-right Point right

Wait Indicator

                    
Use a wait indicator to show a process is in progress that will occur for an indefinite amount 
                        of time by applying the CSS class wait to an element.

                    
Sizing

                    
Adding the large CSS class will result in a larger wait indicator.

                    
Centering

                    
Adding the center CSS class will make the wait to be centered in 
                        the parent element. The parent must be positioned.
